The `labelPosition` hint indicates where the label for a property to be rendered: typically to the left or the top, or sometimes none:

* for properties, the hint can be specified using either the `@PropertyLayout` annotation, or a meta-annotation, or using the `Xxx.layout.xml` layout file.

* for parameters, the hint can be specified using either the `@ParameterLayout` annotation, or a meta-annotation.

Some data types also support the label being rendered to the right (support for this is shown on each of the demo pages for each of the data types).

== Layout File

For properties (not parameters), the CSS class hint can be specified using the `Xxx.layout.xml` file:

* property definition:
+
[source,java,indent=0]
----
include::PropertyLayoutLabelPositionVm.java[tags=layout-file]
----
<.> `labelPosition` hint is _not_ present.

* layout file snippet:
+
[source,xml,indent=0,tabsize=4]
----
<cpt:property id="propertyUsingLayout" labelPosition="TOP"/>
----

One advantage of the layout file over the annotation is that the layout file is dynamic: it can be modified and the changes reloaded in a running application.

TIP: It is sometimes necessary to invoke the `Rebuild Metamodel` action to see this take effect.

* `isis.applib.annotation.property-layout.label-position` configuration property for properties, and
* `isis.applib.annotation.parameter-layout.label-position` configuration property for parameters.
